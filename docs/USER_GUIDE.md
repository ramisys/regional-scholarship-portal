# User Guide

This guide explains how to use the Regional Scholarship Portal as a student or coordinator. It focuses on everyday tasks, what each screen is for, and how the main workflow fits together.

## 1. What this application is for

The Regional Scholarship Portal helps applicants submit scholarship requests and helps coordinators review, approve, or reject them.

It provides:
- Student registration and login
- Scholarship application creation and tracking
- Document upload for supporting files
- Coordinator dashboards and review tools
- Status tracking across the application lifecycle

## 2. Getting started

### Access the portal
1. Open the frontend in your browser.
2. Use the login page if you already have an account.
3. Use the registration page if you are new.

### Choose the correct role
When you register, select one of the available roles:
- Student — for applying for scholarships and uploading documents
- Coordinator — for reviewing applications and making decisions

If you sign in with the wrong role, you may not be able to access the right dashboard.

## 3. Student guide

### 3.1 Create your account
1. Go to the registration page.
2. Enter your name, email, password, and role.
3. Submit the form.
4. After successful registration, you are redirected to your dashboard.

### 3.2 Sign in and understand your dashboard
After logging in, the student dashboard gives you a quick overview of:
- how many applications you have submitted
- how many are pending
- how many were approved or rejected
- recent activity related to your application history

Use the dashboard as your starting point for all student tasks.

### 3.3 Start a new scholarship application
From the student area:
1. Open the application page.
2. Complete the required fields in the form.
3. Save your progress as a draft if you need to return later.
4. Submit the final application when all information is complete.

The application form usually covers:
- personal details
- contact information
- education history
- any supporting notes requested by the scholarship program

### 3.4 Save a draft before submitting
Saving a draft is useful when:
- you are gathering documents
- you want to review your details later
- you do not yet have all the required information

A draft keeps your work in progress without sending the application for review.

### 3.5 Upload supporting documents
Documents are often required to support your scholarship request.

When uploading:
- use supported file types such as PDF, JPG, or PNG
- keep file sizes within the platform limits
- make sure the file is readable and clearly named
- upload only documents that are relevant to your application

You can usually upload files from the student documents section and later review what has already been submitted.

### 3.6 Track your application status
After submission, you can monitor the progress of your application from the student applications view.

Typical statuses include:
- Draft — not yet submitted
- Pending — submitted and waiting for review
- Under Review — currently being assessed by a coordinator
- Approved — accepted
- Rejected — not approved

This status panel helps you understand whether action is needed from you or whether the review is in progress.

### 3.7 What to do after submission
After you submit your application:
1. Wait for the coordinator to review it.
2. Check the status page regularly.
3. Upload any missing documents if requested.
4. Contact support if your application is delayed beyond the expected review period.

## 4. Coordinator guide

### 4.1 Create a coordinator account
1. Register using the coordinator role.
2. Sign in with your new account.
3. Open the coordinator dashboard.

### 4.2 Use the coordinator dashboard
The coordinator dashboard gives a summary of:
- total applications received
- pending review items
- approvals made today
- rejections made today
- recent applications that need attention

This screen helps coordinators prioritize work and review the application queue efficiently.

### 4.3 Review applications
From the applications section, you can:
- search for a specific applicant
- filter applications by status or region
- open individual application details
- view supporting documents attached to an application

Use the detail view to check whether the applicant has completed all required fields and provided enough supporting evidence.

### 4.4 Make approval or rejection decisions
When you are ready to act on an application:
1. Open the application details.
2. Review the applicant information and documents.
3. Confirm the application is complete and valid.
4. Approve or reject it using the action controls.

The decision updates the application status and helps the applicant understand the outcome.

### 4.5 Work with document evidence
Coordinators should review uploaded files carefully to confirm that they are:
- relevant to the application
- readable and complete
- aligned with the submitted information

If a file is missing or unclear, the coordinator can request corrections or ask the applicant to resubmit the relevant document.

## 5. Recommended workflow for most users

### For students
1. Register and sign in.
2. Open the student dashboard.
3. Start a new application.
4. Save your work as needed.
5. Upload supporting documents.
6. Submit the application.
7. Monitor its status until a decision is made.

### For coordinators
1. Sign in to the coordinator dashboard.
2. Review the queue of applications.
3. Open a case for detailed inspection.
4. Check applicant details and attached documents.
5. Approve or reject the application.
6. Use the dashboard to continue reviewing the next case.

## 6. Best practices

- Keep your profile information up to date.
- Save drafts often when working on a long application.
- Upload clear, readable files only.
- Check your status page regularly after submission.
- Use the dashboard summaries to focus on the most urgent work.
- Confirm that applicant data matches the uploaded documents before making a decision.

## 7. Troubleshooting tips

### Login problems
- Confirm that your email and password are correct.
- Make sure you are using the correct account type.
- If needed, restart the session and try again.

### Application not submitting
- Check whether required fields are still incomplete.
- Save the form as a draft first and review it carefully.
- Verify that all files are in an accepted format and within the size limit.

### Documents not appearing
- Refresh the page after uploading.
- Confirm the upload completed successfully.
- Check whether the file type and size are supported.

### Dashboard not loading
- Verify the backend API is running.
- Check that the frontend is pointing to the correct API URL.
- Confirm that your account has the expected role permissions.

## 8. Summary

The portal is designed to make scholarship applications simple and organized:
- students apply, upload documents, and track status
- coordinators review, verify, and decide on applications

Following the steps in this guide will help both applicants and coordinators use the platform effectively and avoid common mistakes.
